Paint.IT – Projektdokumentation

1. Projektüberblick

Paint.IT ist ein Fullstack-System zur Verwaltung von Werkstattaufträgen, Benutzern und Filialen mit folgenden Kernfunktionen:

  • Benutzerverwaltung mit Rollen (Admin, Werkstatt, Buchhaltung, etc.)
  • Multi-Filial-Unterstützung (Benutzer können mehreren Filialen zugeordnet werden)
  • Auftragsmanagement: Anlegen, Bearbeiten, Löschen, Abschließen von Aufträgen
  • Abgeschlossene Aufträge werden separat gelistet
  • Fertigstellungsdatum kann bearbeitet werden, alte Werte werden dokumentiert
  • Filter- und Suchfunktionen für Aufträge
  • Frontend: React + TypeScript, Backend: Node.js (Express, ES Modules), Datenbank: MariaDB/MySQL
  • Authentifizierung via JWT

2. Setup-Tutorial (neuer Server)

Voraussetzungen

  • Node.js (empfohlen: v18+)
  • npm (wird mit Node installiert)
  • MariaDB oder MySQL-Server (empfohlen: MariaDB 10.5+)
  • Git (optional, für Cloning)

Schritt-für-Schritt-Anleitung

  1. System aktualisieren & Grundpakete installieren
    sudo apt update && sudo apt upgrade -y
    sudo apt install -y git curl build-essential
  2. Node.js & npm installieren (empfohlen: Node 18 LTS)
    curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
    sudo apt install -y nodejs
    Prüfen: node -v & npm -v
  3. MariaDB installieren & einrichten
    sudo apt install -y mariadb-server mariadb-client
    Starte und sichere MariaDB (folge den Anweisungen und setze ein root-Passwort): 
    sudo systemctl enable --now mariadb
    sudo systemctl start mariadb
    sudo mysql_secure_installation
  4. Datenbank & User anlegen
    sudo mysql -u root -p
    CREATE DATABASE paintit CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    CREATE USER 'paintituser'@'localhost' IDENTIFIED BY 'deinpasswort';
    GRANT ALL PRIVILEGES ON paintit.* TO 'paintituser'@'localhost';
    FLUSH PRIVILEGES;
    EXIT;
  5. Projekt klonen
    git clone https://github.com/N0C2/paintit.git paintit
    cd paintit
  6. Backend einrichten
    cd server
    npm install
    .env Datei anlegen (server/.env): 
    DB_HOST=localhost
    DB_USER=paintituser
    DB_PASSWORD=deinpasswort
    DB_NAME=paintit
    JWT_SECRET=dein_geheimes_token
    Datenbank-Tabellen erstellen: 
    npm run setup
  7. Frontend einrichten
    cd ../client
    npm install
    npm run build
  8. Anwendung starten

    Für den Produktivbetrieb wird das `install-and-start.sh`-Skript empfohlen:

    cd ..
bash install-and-start.sh

    Für die lokale Entwicklung:

    # Terminal 1 (im 'server' Ordner)
npm start

# Terminal 2 (im 'client' Ordner)
    npm run dev

    Öffne anschließend http://localhost:5173 im Browser.

Wichtige Abhängigkeiten (server)

  • express, cors, dotenv, mysql2, bcrypt, jsonwebtoken, crypto, fs/promises

Wichtige Abhängigkeiten (client)

  • react, react-dom, react-router-dom, typescript, vite

Hinweise

  • Für Produktivbetrieb sichere Passwörter und Umgebungsvariablen verwenden!
  • MariaDB/MySQL-User benötigt Rechte zum Anlegen von Tabellen und Datenbanken.
  • Frontend und Backend können auf unterschiedlichen Servern laufen, API-URL ggf. anpassen.
